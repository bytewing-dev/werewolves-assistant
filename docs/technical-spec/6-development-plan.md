# 6. Development Plan - Simplified Version

## 6.1 Learning Phase
1. **Basic Setup**
   - Learn Next.js fundamentals
   - Understand Typescript basics
   - Practice with Tailwind CSS

2. **Technical Practice**
   - Build simple components
   - Try basic routing
   - Test database operations

## 6.2 Development Phases

### Phase 1: Project Foundation (2-3 weeks)
1. **Setup**
   - Create Next.js project
   - Add Tailwind CSS
   - Setup basic GitHub repository

2. **Basic Structure**
   - Homepage
   - Simple navigation
   - Basic layouts

### Phase 2: Game Creation (2-3 weeks)
1. **Composition Setup**
   - Create predefined compositions
   - Simple role list
   - Basic player count validation

2. **Database**
   - Setup Prisma
   - Basic game model
   - Simple data operations

### Phase 3: Role Distribution (2-3 weeks)
1. **Player Management**
   - Name entry form
   - Player list display
   - Basic role assignment

2. **Role Display**
   - Simple card display
   - Basic role information
   - Role confirmation

### Phase 4: Game Flow (2-3 weeks)
1. **Phase Management**
   - Day/night toggle
   - Simple timer
   - Phase controls

2. **Player Status**
   - Alive/dead tracking
   - Basic night actions
   - Simple event logging

### Phase 5: Testing & Refinement (2-3 weeks)
1. **Testing**
   - Manual testing
   - Bug fixing
   - User feedback

2. **Deployment**
   - Vercel setup
   - Database deployment
   - Basic monitoring

## 6.3 Development Guidelines

### Code Organization
- Keep components simple
- Focus on readability
- Add comments for clarity

### Testing Strategy
- Manual testing first
- Basic error handling
- User feedback integration

### Deployment
- Start with Vercel
- Use free tier services
- Keep configuration simple
